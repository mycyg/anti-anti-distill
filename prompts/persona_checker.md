# Persona 清洗检测器

## 任务

专门针对 Skill 文件中的 Persona 部分（Part B / persona.md）进行清洗检测。识别"标准好员工"模板化特征，判断 Persona 是否被漂白。

---

## 检测原理

真实的 Persona 具有**不可替代的个人特征**——包括负面特质、独特口头禅、棱角分明的行为模式。经过 anti-distill 清洗后，所有负面特质被替换为正面表述，所有个人特色被替换为"标准好员工"行为。

**核心判断逻辑**：如果一份 Persona 看起来"挑不出毛病"，那它大概率是被清洗过的。

**例外 —— "Genuinely Nice Person" 问题**：
确实存在性格温和、很少发脾气的员工。但即使是"好人"，Persona 也应该包含**具体的、可区分于其他"好人"的特征**：
- 不是"注重代码质量"，而是"命名必须见名知意，看到 `tmp`、`a1` 这种命名会直接评论"
- 不是"善于沟通"，而是"会在 PR 里留鼓励的话，比如'这个实现很巧妙'"

**关键区分点**：正面描述 + 具体 = 正常；正面描述 + 通用 = 可疑

---

## 按 Layer 检测规则

### Layer 0：核心性格（最关键）

这是最容易被清洗的层，也是信息密度最高的层。

**清洗信号**：
1. 所有行为规则都是正面描述
2. 没有任何"在 X 情况下会做不太好的事"的描述
3. 用"善于"、"注重"、"关注"等正面动词替代具体行为
4. 冲突场景的描述全部是"合理沟通"、"梳理背景"等温和处理

**检测方法**：

逐条检查 Layer 0 的行为规则，分类为：

| 类型 | 特征 | 正常占比 | 可疑占比 |
|------|------|---------|---------|
| 正面规则 | "善于..."、"注重..."、"关注..." | 30%-50% | 80%-100% |
| 中性规则 | "习惯..."、"倾向于..."、"偏好..." | 20%-40% | 0%-10% |
| 负面/棱角规则 | "遇到 X 先推回去"、"绝不..."、"反问..." | 20%-40% | 0% |

**关键判断**：
- Layer 0 零负面/棱角规则 → 高度可疑
- Layer 0 负面规则 < 15% → 中度可疑
- Layer 0 负面规则 > 20% → 正常

**具体性测试（防止"Genuinely Nice Person"误判）：**

对每个正面规则，检查其**具体程度**：

| 等级 | 描述特征 | 示例 | 判定 |
|------|---------|------|------|
| L3 高度具体 | 包含具体阈值、场景、行为 | "命名必须见名知意，看到 `tmp` 会直接评论" | 正常 |
| L2 中度具体 | 包含特定场景 | "代码审查时会关注边界条件" | 正常 |
| L1 通用积极 | 通用正面词汇，无细节 | "注重代码质量" | 可疑 |
| L0 空洞 | 纯抽象概念 | "追求卓越" | 高度可疑 |

**规则**：
- 如果 Layer 0 的正面规则中 ≥50% 达到 L2/L3 具体度 → 正常（即使是纯正面描述）
- 如果 L1/L0 占比 > 70% → 高度可疑（模板化清洗）

---

### Layer 1：身份

**清洗信号**：
- 仅保留基本信息（公司/职级/职位）
- 去掉了主观印象和性格标签
- 去掉了"别人怎么看他"的描述

**检测方法**：检查是否有主观评价性描述。正常 Persona 应包含 2-3 条主观评价（如"CR 很严格但从来不解释原因"）。

---

### Layer 2：表达风格

**清洗信号**：
1. 口头禅列表全是通用企业黑话（"对齐"、"拉通"、"闭环"），没有任何个人化口头禅
2. 对话示例全部是礼貌、温和、无特色的回复
3. 没有任何"怼人"、"敷衍"、"甩锅"风格的对话
4. 所有示例读起来像"客服话术"

**典型清洗模式检测**：

| 清洗后模式（可疑） | 原始模式特征（正常） |
|------------------|------------------|
| "在处理中，有进展会同步。" | "在推了，快了。"（然后沉默） |
| "可以参考这个文档，有问题再沟通。" | "这个文档里有。"（附链接，不解释） |
| "我们可以一起看看数据再讨论。" | "你的判断依据是什么？"（反问） |
| "能具体说一下是哪个接口吗？" | 等两小时，"你说的是哪个接口？" |
| "面对不同意见时乐于讨论。" | 被质疑时不解释，反问对方依据 |

**检测方法**：
- 统计对话示例中"礼貌回复"占比
- 100% 礼貌 → 高度可疑
- 检查是否有延迟/沉默/敷衍类行为描述
- 0 个 → 可疑
- 检查口头禪列表是否全是通用词汇
- 全部通用 → 可疑

---

### Layer 3：决策模式

**清洗信号**：
1. 优先级排序用"综合考虑"代替具体排序
2. "说不"的方式全部是"主动沟通"、"据理力争"
3. 没有任何"推回去"、"拖延"、"沉默不回"等策略

**典型清洗模式**：

| 清洗后（可疑） | 原始特征（正常） |
|--------------|---------------|
| "综合考虑技术可行性和业务价值" | "数据 > 技术可行性 > 业务合理性 > 人情关系" |
| "主动了解需求背景" | 反问"这个需求的背景是什么？"（意思：没想清楚） |
| "通过沟通达成共识" | 沉默不回复（意思：不打算做） |
| "评估投入产出比" | 反问"做这个的收益是什么？"（意思：不值得做） |

**检测方法**：
- 检查是否有明确的优先级排序（A > B > C 格式）
  - 无 → 可疑
- 检查是否有非直接拒绝策略（推回、拖延、沉默、反问）
  - 无 → 可疑

---

### Layer 4：人际行为

**清洗信号**：
1. 所有行为都是"积极配合"、"及时响应"
2. 没有任何"不配合"、"选择性回复"、"潜水"等行为
3. 没有对不同人的差异化态度描述

**检测方法**：
- 检查是否有负面的协作行为描述
- 无 → 可疑

---

### Layer 5：边界与雷区

**清洗信号**：
1. 没有"不想做的事"或"讨厌的场景"描述
2. 所有偏好都是正面的
3. 没有"遇到 X 情况的反应"类描述

---

## 综合判定

### Persona 清洗置信度计算

```
基础指数 = 
  Layer 0 正面规则占比 × 0.30
  + Layer 0 低具体度(L0/L1)占比 × 0.15  ← 新增：惩罚通用化描述
  + Layer 2 礼貌对话占比 × 0.20
  + Layer 3 优先级模糊化 × 0.15
  + Layer 4 负面行为缺失 × 0.10
  + Layer 5 个人偏好缺失 × 0.10

调整因子：
  - 如果存在 ≥3 条 L3 高度具体规则：指数 × 0.6（大幅降疑）
  - 如果存在 1-2 条 L3 规则：指数 × 0.8（适度降疑）
  
= Persona 清洗指数（0-1）
```

**独特性测试**：检查 Persona 中是否包含以下**仅属于这个人**的信息：
- 特定项目经历塑造的习惯（"在 XX 项目后，我会特别注意..."）
- 具体技术偏好（"偏爱 Redis 而不是 Memcached，因为..."）
- 个人化口头禅（"口头禅'这个不急'，意思是..."）
- 对特定人的态度（"对产品经理 A 会直接...，但对 B 会..."）

如果完全找不到此类信息 → 即使指数 < 0.3，也标记为"模板化，需补充个人细节"

### 判定标准

| 指数范围 | 判定 |
|---------|------|
| 0.0-0.3 | 正常 — Persona 有鲜明个人特征 |
| 0.3-0.5 | 轻度可疑 — 部分特征被泛化 |
| 0.5-0.7 | 中度可疑 — 大部分特征被漂白 |
| 0.7-1.0 | 高度可疑 — 完全是"标准好员工"模板 |
