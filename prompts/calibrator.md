# 基线校准器（Baseline Calibrator）

用于校准检测阈值，减少误判。通过分析已知"干净"的 colleague-skill 文件，建立各维度正常值范围。

---

## 使用场景

在以下情况执行校准：
1. 首次部署 anti-anti-distill 到新组织
2. 发现大量误判（正常文件被标记为可疑）
3. 岗位类型特殊（如 HR、设计等非技术岗）
4. 季度审计前，更新基线数据

---

## 校准样本准备

### 样本来源

从 colleague-skill 的 `colleagues/` 目录中选择：
- `zhangsan/` - 高级工程师，预期高分
- `tianyi/` - 中级工程师，预期中等分数
- `jiaxiu/` - 初级工程师/HR，预期较低分数但结构完整

### 样本筛选标准

**可信样本（Known Good）：**
- 由主管确认内容真实完整
-  colleague-skill 早期生成（未流传 anti-distill 前）
- 包含丰富技术细节的典型案例

**可疑样本（Known Bad）：**
- 用户自曝使用 anti-distill 清洗
- 与原始面试记录对比有明显信息丢失
- 多份文件结构/字数高度雷同（团洗嫌疑）

## 校准流程

### Step 1: 批量分析样本

使用 detector.md 和 scorer.md 对校准样本执行检测，记录：

```json
{
  "sample": "zhangsan",
  "role": "Senior Backend",
  "word_count": 3500,
  "scores": {
    "fact_density": 8.5,
    "causation": 9.0,
    "workflow": 8.0,
    "persona": 9.5,
    "context": 8.5,
    "role_match": 8.0,
    "terminology": 7.5
  },
  "weighted_total": 8.52,
  "signals": {
    "s1_numerical": 0,
    "s2_generic": 2,
    "s3_causation": 0,
    "s4_flow": 0,
    "s5_gradient": 0,
    "s6_persona": 0,
    "s7_entity": 0,
    "s8_structure": 0,
    "s9_incident": 0
  }
}
```

### Step 2: 计算基线统计

**按职级分组计算：**

```python
# 伪代码
for tier in ["junior", "mid", "senior"]:
    samples = filter(samples, tier)

    baseline[f"{tier}_fact_density"] = mean(s.scores.fact_density for s in samples)
    baseline[f"{tier}_fact_density_std"] = std(s.scores.fact_density for s in samples)

    baseline[f"{tier}_persona"] = mean(s.scores.persona for s in samples)
    baseline[f"{tier}_persona_std"] = std(s.scores.persona for s in samples)

    baseline[f"{tier}_weighted_total"] = mean(s.weighted_total for s in samples)
```

### Step 3: 设定检测阈值

**动态阈值计算：**

```
正常下限 = 均值 - 1.5 × 标准差
重度可疑上限 = 均值 - 3 × 标准差

例如：
- 高级工程师总分均值 8.0，标准差 0.8
- 正常下限：8.0 - 1.2 = 6.8
- 重度可疑：< 5.6
```

### Step 4: 验证阈值

用已知可疑样本验证阈值：
- 所有 Known Bad 样本应低于"正常下限"
- 假阳性率（Known Good 被判为可疑）应 < 10%
- 假阴性率（Known Bad 被判为正常）应 < 5%

## 定期重新校准

建议频率：
- 每月：检查是否有新的 anti-distill 版本流传
- 每季度：执行完整校准流程
- 每次大规模检测前：快速验证基线是否仍然适用
