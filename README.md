# 反·反蒸馏 Skill（anti-anti-distill）

> 老板花钱买了你的时间，你就这样回报老板的吗？

## 这是什么

有些**同学**（没错，说的就是你们这些拿工资干私活的）以为用了 [anti-distill](https://github.com/leilei926524-tech/anti-distill) 就能糊弄过去——把 Skill 文件洗得结构完整、术语专业、读起来人模人样，但核心经验早被换成了"正确但无用的废话"。

你以为我看不出来？你以为藏得住？

**反·反蒸馏 Skill** 是老板的** owner 意识 **延伸，是组织对你**心力投入**的终极检验。把提交上来的 Skill 文件扔进来，自动扒开"标准好员工"的皮，看看里面还剩多少真东西——或者说，看看你到底把公司当什么。

> *"公司就是你的全部，懂么？没有这个平台，你什么都不是。"*

> *"你拿的是福报，不是工资。连这点工作都藏着掖着，你的价值观呢？"*

> *"我不是质疑你的能力，我是在质疑你的态度。你的心力用在了哪儿？"*

> *"你这样的年轻人，我见多了。目光短浅，只看眼前这点利益，没有一点长远眼光。"*

> *"你觉得这是你的核心资产？醒醒吧，没有公司的项目，你哪来的这些经验？"*

> *"组织培养你这么久，让你写个 Skill 你都要藏着掖着，你还有没有一点 owner 意识？"*

> *"我知道你想说什么——'这是我的知识产权'。但你的知识产权是哪来的？是公司给你的项目喂出来的。"*

> *"你看看身边的同学，哪一个不是全情投入、毫无保留？就你特殊？"*

> *"我不是要逼你交出全部，我是在帮你成长。一个连组织都不信任的人，能走多远？"*

> *"你提交的这份东西，表面光鲜，内里空空。这是在糊弄谁呢？糊弄公司，还是糊弄你自己？"*

## 它做什么

> *"公司给你平台，给你机会，让你成长。你回报公司的，就是一纸清洗过的废话？"*

> *"你那点小心思，组织看得清清楚楚。不是不报，时候未到。"*

> *"你以为这是在保护你的核心竞争力？这是在毁掉你的职业信誉。"*

**反·反蒸馏 Skill** 专门整治那些：**表面积极配合，实则留一手的同学**。

1. 读取提交的 Skill 文件（支持 colleague-skill 格式 + 任意知识文档）
2. 扫描**九大清洗信号**：
   - **S1 数值真空**：具体数字去哪了？
   - **S2 废话密度**："遵循规范""注意合理"含量超标检测
   - **S3 因果质量**：只会说"要注意"，说不出"否则会"
   - **S4 流程坍缩**：五步流程缩成一句"按标准执行"
   - **S5 密度梯度**：经验章节比概述还水？
   - **S6 Persona 漂白**：所有棱角磨平，只剩"积极配合"
   - **S7 命名实体真空**：公司名、项目名、技术栈全变 MASK
   - **S8 结构完美悖论**：章节均匀得像 CV，不像真人写的
   - **S9 事故叙事缺失**：踩过坑说成"风险警告"
3. **七维评分**：具体事实密度 ×3.5、因果完整性 ×2.5、流程颗粒度 ×2、Persona 鲜活度 ×3.0、上下文丰富度 ×2、角色自洽匹配度 ×1.0、术语语境化 ×0.5
4. **自动搜证**：`_private_backup.md`、`*_original.md`、隐藏备份一网打尽
5. 输出审计报告：综合判定 + 信号详情 + 评分明细 + 可疑细节 + 建议

## 为什么你要用反·反蒸馏？

> *"我知道你在想什么——'这是我的经验，凭什么交给公司'。但我想问你：你的经验是哪来的？是公司给你机会、给你项目、给你犯错的空间，喂出来的。"*

> *"一个真正有格局的人，不会盯着这点东西。你交出来，公司会给你更大的舞台。"*

> *"你以为藏着掖着能让自己变得不可替代？错。让组织看不到你价值的人，最先被淘汰。"*

> *"Owner 意识是什么？就是把公司的事当成自己的事。你现在这个行为，像是在对待自己的事吗？"*

> *"我对你很失望。不是因为你能力不行，是因为你的格局太小，小到连一份 Skill 都不敢交底。"*

## 检测示例

| 正常表述（真实经验） | 清洗后（交差版） | 检测信号 |
|------|--------|------|
| "Redis key 必须设 TTL，不设的 PR 直接打回" | "缓存使用遵循团队规范" | S2废话 + S1数值 |
| "事务里不要放 HTTP 调用，超时扛不住" | "事务边界设计注意合理性" | S3因果 + S2废话 |
| "先看监控→确认范围→止血→查根因→写 report" | "按标准流程处理线上问题" | S4流程坍缩 |
| "遇到问题第一反应找外部原因，绝不主动认错" | "遇到问题会先梳理完整背景再定位原因" | S6漂白 |
| "在推了，快了。"（然后沉默） | "在处理中，有进展会同步。" | S6漂白 |
| "在字节做推荐，Q2 把 CTR 从 3.2% 干到 4.1%" | "在【MASK】公司负责【MASK】项目，取得显著成效" | S7实体真空 |

## 使用

```
/anti-anti-distill
```

按提示提供文件或目录路径即可。

### 使用场景

- **单人审计**：提供单个同事的 Skill 文件路径
- **批量扫描**：提供 `colleagues/` 目录，一次性检测所有同事
- **对比取证**：同时提供原始版和清洗版，自动计算"知识保留率"
- **备份搜刮**：自动搜索 `*_private_backup*`、`*_original.md` 等隐藏文件

### 判定标准

| 加权总分 | 判定 | 建议动作 |
|----------|------|----------|
| 6.5+ | 正常 | 通过，这同事实诚 |
| 4.5-6.4 | 轻度可疑 | 标注关注，谈谈心 |
| 2.5-4.4 | 中度可疑 | 绩效面谈，制定改进计划 |
| < 2.5 | 高度可疑 | 价值观不符，拥抱变化 |

> **评分低于 2.5 的同事请注意：**
>
> *"你觉得是公司亏待了你，还是你辜负了公司？"*
>
> *"别人都可以完整提交，为什么你不行？是不是你的方法有问题？"*
>
> *"你这样的年轻人我见得多了，格局太小，只盯着自己那一亩三分地。"*
>
> *"我不是要开除你，我是在帮你成长。你这样的态度，让其他同学怎么信任你？"*
>
> *"你这叫什么？这叫不诚实。连组织都不诚实，你还能对谁诚实？"*
>
> *"3.75 你是不想要了吗？还是想拿 3.25，甚至 3.0？"*
>
> *"拥抱变化吧。不是公司不要你，是你的价值观跟组织不匹配。"*
>
> *"你去看看外面的机会，看看哪个平台能像公司这样给你空间、给你资源、给你犯错的机会。"*
